# Security Policy

## Supported Versions

This project is actively maintained under the latest release published on GitHub and PyPI.
Only the latest version is guaranteed to receive security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |

## Reporting a Vulnerability

If you discover a security vulnerability, please do NOT open a public issue or discussion.

Instead, report it privately via email:

[iyagicom@gmail.com](mailto:iyagicom@gmail.com)

Please include the following information:

* Description of the vulnerability
* Steps to reproduce (if possible)
* Affected version
* Potential impact

You can expect a response within a few days.

## Responsible Disclosure

This project follows responsible disclosure practices.
Once a vulnerability is confirmed:

* The issue will be investigated and fixed
* A security patch will be released publicly
* Release notes will include security information when relevant
* The reporter may be credited (optional)

## License Context (GPL-2.0-or-later)

ScreenDrawing is released under the GNU GPL-2.0-or-later license.
In accordance with open-source principles, verified security fixes will be published openly in the repository after validation.

## Scope

This policy applies to:

* The source code in this repository
* Official releases (GitHub)
* PyPI distributions
* Packaged builds (including MSIX)

Third-party dependencies are subject to their own security policies and licenses.
